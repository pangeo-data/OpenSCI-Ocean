# Review Checklist / 审查清单

每个项目在 D2（人类审查）和 D3（内部评审）阶段使用本清单。审查者在 PR 中逐项打勾。

---

## D2: 人类审查交付稿（物理把关）

由 Domain Expert 完成，重点检查 AI 初稿的科学正确性。

### 科学内容
- [ ] 核心科学问题清晰、重要、有新颖性
- [ ] 研究假设明确可检验
- [ ] 物理机制解释正确（无因果倒置、无违背基本物理定律）
- [ ] 数值结果量级合理（速度、温度、波高、谱斜率等）
- [ ] 统计分析方法恰当（显著性检验、误差估计等）
- [ ] 结论由数据充分支撑，无过度推断

### 文献引用
- [ ] 每条参考文献均已通过 Google Scholar / Crossref 核验存在
- [ ] 无编造或张冠李戴的引用
- [ ] 引用覆盖了该方向的关键文献
- [ ] 未遗漏直接相关的最新重要文献

### 数据与方法
- [ ] 数据来源清楚标注（数据集名称、版本、时间范围、空间范围）
- [ ] 数据处理步骤可复现
- [ ] 分析代码可运行，结果一致
- [ ] 数据源链接有效可访问

### 图表
- [ ] 图表清晰、标注完整（坐标轴标签、单位、色标、图例）
- [ ] 图表与正文描述一致
- [ ] 配色方案合理（色盲友好、物理含义正确）
- [ ] 分辨率满足期刊要求

---

## D3: 内部评审定稿（模拟审稿）

由另一位参与者（非该项目主要贡献者）扮演审稿人角色完成。

### 整体评价
- [ ] 论文整体逻辑连贯（Introduction → Methods → Results → Discussion 链条完整）
- [ ] 创新点突出，与已有文献有清晰区分
- [ ] 适合目标期刊的定位和读者群

### 写作质量
- [ ] 摘要准确概括全文要点
- [ ] Introduction 从大到小收束，清楚引出科学问题
- [ ] Methods 详略得当，关键步骤可复现
- [ ] Results 与 Discussion 分离清晰（如期刊要求）
- [ ] Discussion 有深度，讨论了局限性和未来方向
- [ ] 语言流畅、无明显语法错误

### AI 痕迹检查
- [ ] 无 AI 典型套话（"it is worth noting"、"delve into"、"comprehensive framework" 等）
- [ ] 无空洞的过渡句或无信息量的总结段
- [ ] 行文风格一致，无突兀的语气切换
- [ ] 无不必要的 hedge（"may potentially"、"could possibly" 等）

### 投稿准备
- [ ] 格式符合目标期刊要求（模板、字数、图表数量限制）
- [ ] Cover Letter 突出创新点和重要性
- [ ] 作者信息、通讯信息、ORCID 完整
- [ ] AI 使用声明已按期刊要求撰写
- [ ] 数据可用性声明（Data Availability Statement）已撰写
- [ ] CRediT 作者贡献声明已准备

---

## 使用方法

1. 审查者 Fork 对应的 PR
2. 逐项检查并在评论中打勾 / 提出修改意见
3. 所有必选项通过后，Approve PR
4. 如有重大问题，Request Changes 并具体说明
