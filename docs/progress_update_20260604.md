# 10个人，10个方向，一周一篇——我们开始了

上一篇推文发出不到48小时，报名人数超出预期。

目前已有10余位同行加入工作组。硕士、博士、博后、青年教师都有，方向涵盖卫星高度计、物理海洋、海气相互作用、极地遥感。来自不同学校和机构，共同点只有一个：对这件事真的好奇。

本项目是一个探索性质的科研协作实验，完全自发，由制造国产新一代低成本高精度小型化 GNSS 波浪潮汐浮标的青岛安海发起，邀请公众号粉丝参加。

愿景上篇讲过了。这篇讲进展，讲接下来怎么干。

---

## 框架搭好了

过去两天，协作框架在 GitHub 上搭建完毕（仓库：github.com/pangeo-data/OpenSCI-Ocean）。

**10个独立项目，并行推进。** 仓库分成 P01 到 P10 十个子项目，每个对应一个研究方向、一个科学问题。项目之间互不依赖，各自推进，各自署名。你可以只参与一个，也可以同时跟好几个。

**每个项目的节奏：**

- D0：选题确认，AI 快速文献调研，判断新颖性——半天到一天
- D1：AI 生成初稿，包括文献综述、数据分析代码、图表和全文——一天
- D2：人类审查，物理把关，纠偏修改——两到三天
- D3：内部模拟评审，交叉审查，定稿——约三天
- D4：格式检查，Cover Letter，投稿

选题到投稿，大约一周。

这个速度不是拍脑袋定的。AI 负责文献检索、代码编写、初稿生成这些可以批量完成的工作，人类只做 AI 做不好的事：判断物理机制对不对，检查引用是不是真的，决定一个结论能不能立住。

## 怎么分工

**每个项目 2 到 3 人一组，AI 是主力。**

不是十几个人挤在一个项目里开会，而是小团队快速行动。核心组两三个人，负责把控方向、审查结果、做最终判断。文献检索、代码编写、数据处理、初稿生成这些苦活累活，交给 AI。

人做人该做的事——提出好问题，判断物理上通不通，把关结论。

AI 做 AI 擅长的事——搬运、计算、写作、不知疲倦地迭代。

两三个人加一个 AI，产出不输一个传统课题组。

## 规则初稿

协作制度的初稿已写好，放在 GitHub 仓库里，欢迎参与者提意见。几个方向性的原则：

**署名按实际贡献来。** 参照 CRediT 国际标准，每个项目独立署名。提出关键科学问题、做出核心物理判断、提供独家数据、资助项目经费或承担版面费——这些贡献不一定体现在代码提交里，但同样重要。GitHub 记录是参考，不是唯一标尺。

**AI 使用如实声明。** 每个项目记录 AI 交互日志，投稿时在 Methods 部分写清楚 AI 参与了什么、到什么程度。不回避，不夸大。

**几条底线：** 文献零幻觉——AI 给出的每一条引用，必须人工核验存在。代码可复现——分析脚本上传 GitHub，任何人能跑出相同结果。仓库不存原始数据，只存数据链接和下载脚本。

这些规则还在打磨，正式版会在第一个项目启动前定下来。

## 全过程保留

这个项目和一般科研项目最大的不同：全过程透明。

每个项目目录有四个版本——AI 初稿、人类审查稿、内部评审定稿、最终投稿版。每一版都留在 Git 历史里。从 AI 写下的第一个字到最终投稿，每一步修改都可以回溯。

AI 贡献了多少，人改了多少，diff 里看得清清楚楚。

这也是我们想回答的问题：AI 做科研，真实的边界到底在哪？不靠嘴说，靠记录。

## 但这些都不是最重要的

框架、制度、流程、底线——这些保障项目能转起来。重要，但不是我做这件事的原因。

报名的人里，有人想发论文毕业，有人想攒成果评职称。这些都是真实的需求，完全正当。项目能帮上忙，我很高兴。

发论文，拿学位，评教授。都好。

但我知道，很多人走进海洋科学这个行当，最初不是因为这些。

小时候第一次在电视上看到蓝色的地球，第一次翻开一本讲海洋的书，第一次听说马里亚纳海沟有一万米深——那时候没人想过影响因子，没人算过一区二区。想法很简单：

**长大了当科学家。**

后来我们真长大了。读了硕士、博士，进了实验室，学会写代码、跑模型、投稿件。也学会数论文、看 IF、算 h-index。不知道从哪天起，"做科研"变成了"发文章"，探索变成了生产，好奇心被压在了绩效考核下面。

我不批判这个系统。每个人都在里面找自己的路，没有对错。

但我想试试，能不能留一小块地方，回到最开始的样子。

不为发表而发表。就是真的想搞明白一件事：海洋里的涡旋怎么生成的？SWOT 卫星拍到的精细结构到底在说什么？冰盖底下的海水在干什么？AI 这把新工具，能不能帮我们更快、更深地靠近答案？

发在 Nature 子刊，当然好。

发不了，但过程中真搞懂了一些东西，也值。

要是拿了诺贝尔奖——我请全组吃饭。
